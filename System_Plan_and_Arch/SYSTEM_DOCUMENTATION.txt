                            1. EXECUTIVE SUMMARY
================================================================================

This graduation project presents a comprehensive Knowledge Graph system for
Egyptian criminal law. The system leverages cutting-edge technologies including:

• Graph RAG (Retrieval-Augmented Generation) for legal knowledge retrieval
• Neo4j knowledge graphs for structured legal representation
• LLM-powered entity and relationship extraction
• Semantic chunking for optimal legal text processing

KEY FEATURES:
-------------
✓ Structured representation of Egyptian criminal law
✓ Automated extraction of legal entities and relationships
✓ Graph-based retrieval for legal research
✓ Complete provenance tracking for all legal articles
✓ Semantic search capabilities
✓ Relationship mapping between crimes, penalties, and conditions

CRITICAL DESIGN PRINCIPLES:
---------------------------
• ONLY Egyptian criminal law sources
• NO free-text generation without citation
• EXPLICIT provenance for all extracted information
• Complete relationship mapping between legal concepts

--------------------------

PROCESSING:

OUTPUT:

STAGE 2: LEGAL ARTICLE EXTRACTION
----------------------------------
PURPOSE: Preserve legal structure by extracting individual articles

EXTRACTION PATTERN:
• Detect article headers: "المادة X", "مادة X"
• Extract article number
• Preserve article content
• Maintain source attribution

OUTPUT FORMAT:
{
  "text": "المادة 123\n[article content]",
  "metadata": {
    "source": "قانون_العقوبات.pdf",
    "article_title": "المادة 123",
    "article_number": 123,
    "type": "legal_article"
  }
}

---

STAGE 3: SEMANTIC CHUNKING (OPTIONAL)
--------------------------------------
For very long articles, apply semantic chunking to create optimal chunks
for graph construction while preserving legal meaning.

PARAMETERS:
• Chunk size: 1000-1500 characters
• Overlap: minimal (to avoid breaking legal meaning)
• Similarity threshold: 0.65-0.75

---

STAGE 4: EMBEDDING GENERATION
------------------------------
MODEL: Arabic Sentence Transformer
• Model: Omartificial-Intelligence-Space/Semantic-Ar-Qwen-Embed-0.6B
• Device: CPU (for stability) or CUDA (with small batch size)
• Normalization: Enabled

BATCH PROCESSING:
• Small batch sizes (4-8) to avoid memory issues
• Progressive processing with memory clearing
• Text truncation for very long articles

OUTPUT:
• Embedding vectors for each legal chunk
• Vector index for semantic search

---

STAGE 5: GRAPH CONSTRUCTION
----------------------------
TOOL: LangChain LLMGraphTransformer

ALLOWED NODES:
• مادة_قانونية (Legal Article)
• جريمة (Crime)
• عقوبة (Penalty)

ALLOWED RELATIONSHIPS:
• يُعاقب_بـ (is_punished_by): جريمة → عقوبة
• يُحيل_إلى (refers_to): مادة_قانونية → مادة_قانونية


PROCESSING:
1. LLM analyzes each legal chunk
2. Extracts entities (nodes)
3. Identifies relationships
4. Generates graph document

OUTPUT:
• Graph documents with nodes and relationships
• Provenance linking to source text

================================================================================
                       5. KNOWLEDGE GRAPH SCHEMA
================================================================================

NODE TYPES:
-----------
• مادة_قانونية (Legal Article)
  Properties:
    - رقم_المادة (article_number): Integer
    - نص_المادة (article_text): String
    - المصدر (source): String
    - الباب (chapter): String (optional)
    - الفصل (section): String (optional)

• جريمة (Crime)
  Properties:
    - اسم_الجريمة (crime_name): String
    - وصف (description): String
    - المصدر (source): String

• عقوبة (Penalty)
  Properties:
    - نوع_العقوبة (penalty_type): String (e.g., "السجن", "الغرامة")
    - المدة (duration): String (e.g., "5 سنوات")
    - المبلغ (amount): String (for fines)
    - المصدر (source): String

---

RELATIONSHIP TYPES:
-------------------

(جريمة)-[:يُعاقب_بـ]->(عقوبة)
    Crime is punished by this penalty

(مادة_قانونية)-[:يُحيل_إلى]->(مادة_قانونية)
    Legal article refers to another article

STEP 2: CONFIGURE GRAPH TRANSFORMER
------------------------------------
```python
from langchain_experimental.graph_transformers import LLMGraphTransformer
from src.LLMs.Qwen_Model import ArabicModel

llm_ar = ArabicModel()
llm = llm_ar.get_ar_llm()

llm_transformer = LLMGraphTransformer(
    llm=llm,
    allowed_nodes=["مادة_قانونية", "جريمة", "عقوبة", "شرط", "باب", "فصل"],
    allowed_relationships=["يُعاقب_بـ", "يشترط", "يُحيل_إلى", "ينتمي_لـ", "يُشدد_بـ", "يُخفف_بـ"]
)
```

---

STEP 3: EXTRACT GRAPH DOCUMENTS

```python
print("⏳ استخراج Knowledge Graph...")
```

================================================================================
                     7. RETRIEVAL & QUERY SYSTEM
================================================================================

7.1 GRAPH-BASED QUERIES (CYPHER)
---------------------------------

EXAMPLE 1: Find all crimes and their penalties
```cypher
MATCH (crime:جريمة)-[:يُعاقب_بـ]->(penalty:عقوبة)
RETURN crime.اسم_الجريمة, penalty.نوع_العقوبة, penalty.المدة
```

EXAMPLE 2: Find articles in a specific chapter
```cypher
MATCH (article:مادة_قانونية)-[:ينتمي_لـ]->(chapter:باب {رقم_الباب: 2})
RETURN article.رقم_المادة, article.نص_المادة
ORDER BY article.رقم_المادة
```

EXAMPLE 3: Find aggravating conditions for penalties
```cypher
MATCH (penalty:عقوبة)-[:يُشدد_بـ]->(condition:شرط)
RETURN penalty.نوع_العقوبة, condition.نص_الشرط
```

EXAMPLE 4: Find articles that refer to other articles
```cypher
MATCH (article1:مادة_قانونية)-[:يُحيل_إلى]->(article2:مادة_قانونية)
RETURN article1.رقم_المادة, article2.رقم_المادة
```

7.3 HYBRID RETRIEVAL (GRAPH + VECTOR)
--------------------------------------

Combine vector search with graph traversal:

1. Use vector search to find relevant articles
2. Use Cypher to expand context (related crimes, penalties, conditions)
3. Return enriched results

```python
def hybrid_search(query, k=3):
    # Step 1: Vector search
    vector_results = vectorstore.similarity_search(query, k=k)
    
    # Step 2: Graph expansion
    enriched_results = []
    for doc in vector_results:
        article_num = doc.metadata.get('article_number')
        
        # Query Neo4j for related entities
        cypher = f"""
        MATCH (article:مادة_قانونية {{رقم_المادة: {article_num}}})
        OPTIONAL MATCH (article)-[:يتضمن]->(crime:جريمة)-[:يُعاقب_بـ]->(penalty:عقوبة)
        RETURN article, crime, penalty
        """
        
        graph_result = graph_db.query(cypher)
        enriched_results.append({
            'article': doc,
            'related_entities': graph_result
        })
    
    return enriched_results
```
