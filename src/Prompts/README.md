# نظام دعم القرار القضائي للمحاكم المصرية
# Egyptian Judicial Decision Support System

## نظرة عامة

نظام متكامل لدعم القرار القضائي في المحاكم الجنائية المصرية باستخدام وكلاء الذكاء الاصطناعي المتخصصين.

## الوكلاء السبعة

### 1. وكيل التنسيق المركزي (Orchestrator Agent)
**الملف:** `01_orchestrator_agent.py`

**الدور:** منسق مركزي ومدير سير العمل
- استقبال ملفات القضايا
- توزيع المهام على الوكلاء المتخصصين
- تجميع المخرجات المرحلية
- إدارة دورة حياة القضية

### 2. وكيل استيعاب البيانات (Data Ingestion Agent)
**الملف:** `02_data_ingestion_agent.py`

**الدور:** تحويل المستندات غير المهيكلة إلى رسم بياني معرفي
- معالجة المستندات القانونية
- استخلاص الوقائع والأدلة والأشخاص
- بناء رسم بياني في Neo4j
- ربط كل معلومة بمصدرها

### 3. وكيل المراجعة الإجرائية (Procedural Auditor Agent)
**الملف:** `03_procedural_auditor_agent.py`

**الدور:** مراجعة الامتثال الإجرائي وفقاً لقانون الإجراءات الجنائية المصري
- مراجعة إجراءات القبض والتفتيش والاستجواب
- تحديد البطلان (مطلق أو نسبي)
- تقييم أثر البطلان على الأدلة
- تطبيق مبادئ محكمة النقض

### 4. وكيل البحث القانوني (Legal Researcher Agent)
**الملف:** `04_legal_researcher_agent.py`

**الدور:** إجراء البحث القانوني في المصادر المصرية حصراً
- البحث في قانون العقوبات والإجراءات الجنائية
- استخراج المواد القانونية والمبادئ
- جمع السوابق القضائية من محكمة النقض
- تحديد أركان الجريمة والعقوبات

### 5. وكيل تحليل الأدلة (Evidence Analyst Agent)
**الملف:** `05_evidence_analyst_agent.py`

**الدور:** تحليل القوة الإثباتية للأدلة
- ربط الأدلة بالوقائع
- تقييم الاتساق والتعاضد
- رصد التناقضات
- تقييم قوة الإثبات (مقياس من 0 إلى 1)

### 6. وكيل تحليل الدفاع (Defense Analysis Agent)
**الملف:** `06_defense_analysis_agent.py`

**الدور:** تحليل دفوع المتهم الشكلية والموضوعية
- تصنيف الدفوع (شكلية/موضوعية)
- تحديد الأساس القانوني لكل دفع
- تقييم قوة الدفع
- ربط الدفوع بمبادئ محكمة النقض

### 7. وكيل القاضي (Judge Agent)
**الملف:** `07_judge_agent.py`

**الدور:** الموازنة النهائية وإعداد مسودة الحكم المسبب
- عرض الوقائع الثابتة
- مناقشة الجوانب الإجرائية والموضوعية
- تطبيق مبادئ محكمة النقض
- إعداد مسودة حكم مسببة
- تطبيق مبدأ: "الشك يُفسَّر لمصلحة المتهم"

### الاستخدام الأساسي

```python
from judicial_system import JudicialDecisionSupportSystem

# إنشاء نسخة من النظام
system = JudicialDecisionSupportSystem()

# عرض جميع الوكلاء
agents = system.list_all_agents()
for agent in agents:
    print(f"{agent['name']}: {agent['role']}")

# الحصول على prompt وكيل معين
orchestrator_info = system.get_agent_info("orchestrator")
prompt = orchestrator_info['prompt']

# استخدام الـ prompt مع نموذج LLM
```

### استخدام وكيل واحد

```python
from procedural_auditor_agent import (
    get_procedural_auditor_prompt,
    ProceduralAuditResult
)

# الحصول على التعليمات
prompt = get_procedural_auditor_prompt()

# إنشاء نتيجة مراجعة
result = ProceduralAuditResult(
    procedure="تفتيش منزل",
    procedure_id="PROC_001",
    date="2024-01-15",
    status="باطل"
)

result.set_nullity(
    nullity_type="بطلان مطلق",
    defects=["عدم وجود إذن من النيابة"],
    affected_evidence=["E12", "E15"],
    legal_basis=["المادة 91 من قانون الإجراءات"],
    reasoning="التفتيش تم دون إذن وخارج حالة التلبس"
)

# تحويل إلى JSON
json_result = result.to_dict()
```

## المبادئ الأساسية

### 1. الدقة والحياد
- جميع الوكلاء محايدة وموضوعية
- لا تفسير خارج النطاق القانوني المحدد
- الالتزام بالمصادر المصرية حصراً

### 2. الشفافية والتتبع
- كل معلومة مرتبطة بمصدرها
- جميع المخرجات بصيغة JSON موثقة
- إمكانية تتبع كل قرار لأساسه القانوني

### 3. الاحترام الكامل للإجراءات
- تطبيق صارم لقانون الإجراءات الجنائية
- استبعاد الأدلة الباطلة إجرائياً
- احترام حقوق الدفاع

### 4. مبدأ الشك لمصلحة المتهم
- تطبيق صارم للمبدأ الدستوري
- إعلان صريح عن حالات عدم اليقين
- الأصل في الإنسان البراءة
